## Универсальные шаблоны

### Вопросы

* Что такое обобщение?
  * Механизм построения программного кода для некоторого типа с произвольным именем с целью его дальнейшего
преобразования в другой конкретный ссылочный тип.
  * Средства языка для реализации обобщенного программирования
* Что такое закрытый тип?
  * Или параметризованный тип - тип, содержащий фактический параметр типа.
* Что такое открытый тип?
  * Или обобщенный тип - тип, содержащий формальный параметр типа.
* Объясните понятия ковариантности и контрвариантности обобщений.
  * Ковариантность - например, Sub является подтипом Super, то тип массива `Sub[]` является подтипом `Super[]`.
  * Контрвариант (инвариант) - например, для любых двух различных типов `Type1` и `Type2` тип `List<Type1>` не является 
ни подтипом, ни супертипом `List<Type2>`.
* Какие преимущества использования обобщений?
  * Улучшение безопасности и выразительности кода.
  * Уменьшение ошибок времени выполнения, увеличение проверки на этапе компиляции.
* Какие вы знаете типы ограничений для обобщений?
  * Ограничения определяются с помощью extends.
* Что такое ограничение обобщений?
  * При определении универсального типа можно ограничить виды типов, которые могут использоваться клиентским кодом
    в качестве аргументов типа.
* Какие обобщенные коллекции вы знаете?
  * List, ArrayList, Dictionary.

### Задания

**Задание 1**

Создайте класс MyClass<T>, содержащий статический фабричный метод - T factoryMethod(), который
будет порождать экземпляры типа, указанного в качестве параметра типа (указателя места заполнения
типом – Т).

**Задание 2**

Создайте класс MyList<T>. Реализуйте в простейшем приближении возможность использования его
экземпляра аналогично экземпляру класса List<T>. Минимально требуемый интерфейс взаимодействия с
экземпляром, должен включать метод добавления элемента, индексатор для получения значения
элемента по указанному индексу и свойство только для чтения для получения общего количества
элементов.

**Задание 3**

Создайте класс MyDictionary<TKey,TValue>. Реализуйте в простейшем приближении возможность
использования его экземпляра. Минимально требуемый интерфейс взаимодействия с экземпляром,
должен включать метод добавления пар элементов, индексатор для получения значения элемента по
указанному индексу и свойство только для чтения для получения общего количества пар элементов.
